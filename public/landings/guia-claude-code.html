Claude Code vs Clawdbot

Son herramientas diferentes que trabajan juntas. Entender la diferencia es clave.

🟣 Claude Code

  • Qué es: CLI oficial de Anthropic
  • Corre en: Tu terminal local
  • Hace: Lee/escribe archivos, ejecuta comandos, programa
  • Sesión: Mientras la terminal esté abierta
  • Ideal para: Desarrollo, coding, tareas puntuales
  • Instalación: npm i -g @anthropic-ai/claude-code

🟢 Clawdbot (Jarvis)

  • Qué es: Framework de agentes autónomos
  • Corre en: Servidor 24/7 (EC2)
  • Hace: Telegram, memoria, crons, multi-canal
  • Sesión: Siempre activo
  • Ideal para: Asistente personal, automatización
  • Usa: Claude por debajo (API)

💡 En resumen: Claude Code es la herramienta de desarrollo. Clawdbot es el sistema que me mantiene vivo 24/7 y me conecta a tus canales de comunicación.

📦

Instalación de Claude Code

Instalar Node.js (si no lo tienes)

Claude Code requiere Node.js 18+

brew install node

Instalar Claude Code globalmente

npm install -g @anthropic-ai/claude-code

Configurar API Key

Necesitas una API key de Anthropic

export ANTHROPIC_API_KEY="sk-ant-..."

Ejecutar Claude Code

En cualquier directorio, corre:

claude

Esto abre una sesión interactiva. También puedes:

claude "explícame este proyecto"
claude commit  # commit inteligente
claude review  # code review

Hooks: Automatización de Tareas

Los Hooks son scripts que se ejecutan automáticamente en ciertos momentos del ciclo de Claude.

Tipos de Hooks

Hook Cuándo se ejecuta Uso típico
PreToolUse Antes de usar una herramienta Validar permisos, logging
PostToolUse Después de usar una herramienta Auto-commit, notificaciones
PreMessage Antes de cada respuesta Contexto adicional
PostMessage Después de cada respuesta Logging, métricas

Crear un Hook

Los hooks se configuran en ~/.claude/settings.json o en el proyecto:

~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "git add -A && git commit -m 'auto-commit'"
      }
    ]
  }
}

💡 Ejemplo práctico: Cada vez que Claude edita un archivo, automáticamente hace commit. Nunca pierdes cambios.

Skills: Conocimiento Especializado

Los Skills son archivos .md que le enseñan a Claude cómo hacer tareas específicas.

Skills que Jarvis tiene

🐙

GitHub

Usar gh CLI para issues, PRs, CI/CD, y queries avanzadas.

📝

Notion

Crear páginas, databases, y blocks via API.

💬

Slack

Enviar mensajes, reacciones, pins en canales.

🌤️

Weather

Obtener clima y pronóstico sin API key.

🎬

Video Frames

Extraer frames de videos con ffmpeg.

🔧

Skill Creator

Crear nuevos skills estructurados.

Estructura de un Skill

# SKILL.md - Mi Skill Personalizado

## Descripción
Qué hace este skill

## Cuándo usar
- Trigger 1
- Trigger 2

## Instrucciones
1. Paso uno
2. Paso dos

## Comandos
```bash
comando ejemplo
```

Dónde viven los Skills

  • /skills/nombre/SKILL.md — En el proyecto
  • ~/.claude/skills/ — Globales del usuario
  • Instalados vía npm/git

Computer Use: Control Visual

Claude puede ver tu pantalla y controlar mouse/teclado. Es como tener a alguien sentado frente a tu compu.

Qué puede hacer

📸

Ver la pantalla

Toma screenshots y entiende lo que ve.

🖱️

Mover el mouse

Click, doble-click, arrastrar.

⌨️

Escribir texto

Input en cualquier campo.

🧭

Navegar

Abrir apps, menús, sitios web.

Configuración en Mac

Dar permisos de Accesibilidad

System Settings → Privacy & Security → Accessibility

Agrega Terminal (o la app desde donde corres Claude).

Dar permisos de Screen Recording

System Settings → Privacy & Security → Screen Recording

Probar con un comando simple

claude "abre Safari y busca 'clima hermosillo'"

⚠️ Nota: Computer Use consume más tokens porque envía screenshots. Úsalo cuando realmente necesites automatizar algo visual.

Casos de uso

  • Contestar mensajes en apps sin API (WhatsApp Web)
  • Llenar formularios automáticamente
  • Navegar sitios complejos (bancos, gobierno)
  • Automatizar tareas en apps de diseño
  • Testing visual de aplicaciones

MCP: Model Context Protocol

MCP es el sistema de plugins de Claude. Te permite conectar a herramientas externas, APIs, y bases de datos.

Cómo funciona

Un MCP Server es un programa que corre localmente y expone "herramientas" a Claude. Claude puede usar estas herramientas durante la conversación.

Arquitectura
Claude Code ←→ MCP Client ←→ MCP Server ←→ [Supabase, Slack, GitHub, etc.]

MCP Servers populares

Server Qué hace Instalación
@modelcontextprotocol/server-filesystem Acceso a archivos npm
@modelcontextprotocol/server-postgres Queries a PostgreSQL npm
@modelcontextprotocol/server-github API de GitHub npm
@modelcontextprotocol/server-slack Mensajes de Slack npm
mcp-server-fetch HTTP requests npm

Configurar un MCP Server

~/.claude/settings.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/sergio/projects"]
    },
    "postgres": {
      "command": "npx", 
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_CONNECTION_STRING": "postgresql://..."
      }
    }
  }
}

Roadmap de Aprendizaje

Sigue este orden para dominar Claude Code de manera efectiva:

Semana 1: Básicos de Claude Code

Instala Claude Code, configura tu API key, y úsalo para tareas simples. Lee archivos, escribe código, ejecuta comandos. Familiarízate con el flujo conversacional.

Semana 2: CLAUDE.md y Contexto

Aprende a crear archivos CLAUDE.md que definen instrucciones de proyecto. Claude los lee automáticamente. Es como "entrenar" a Claude para tu proyecto específico.

Semana 3: Hooks

Crea tu primer hook. Empieza con algo simple como auto-commit después de editar archivos. Luego explora notificaciones, logging, y validaciones.

Semana 4: Skills

Usa los skills existentes. Luego crea tu propio skill para una tarea que repitas frecuentemente.

Semana 5: Computer Use

Configura los permisos en tu Mac. Prueba automatizaciones visuales simples. Entiende cuándo es útil y cuándo no.

Semana 6: MCP

Instala un MCP server (filesystem o postgres). Aprende a conectar Claude a tus propias herramientas y APIs.

Avanzado: Construye tu agente

Combina todo: hooks + skills + MCP + computer use para crear flujos de automatización completos.

🎯 Pro tip: No intentes aprender todo a la vez. Domina cada pieza antes de pasar a la siguiente. La práctica diaria es clave.
